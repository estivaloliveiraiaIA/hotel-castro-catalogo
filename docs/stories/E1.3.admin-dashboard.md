## Story
**As a** administrador do Castro's Park Hotel,
**I want** um painel web para gerenciar lugares, eventos, roteiros e parceiros,
**so that** eu possa manter o conteúdo do aplicativo atualizado sem precisar de um desenvolvedor.

## Acceptance Criteria
1. Autenticação por senha com token JWT simples (localStorage)
2. Dashboard com contadores: total de lugares, eventos, roteiros e parceiros
3. CRUD completo de Lugares (criar, editar, excluir, ativar/desativar, upload de imagem)
4. CRUD completo de Eventos (com datas, local, imagem, link externo)
5. CRUD completo de Roteiros (com paradas ordenáveis, notas por parada)
6. CRUD completo de Parceiros (logo, categoria, vantagem para hóspedes)
7. Upload de imagens para Supabase Storage
8. Sidebar de navegação com links para cada seção
9. Acessível em `/admin` — bundle isolado via React.lazy (não afeta performance do app principal)

## Dev Notes
- **Autenticação:** `ADMIN_PASSWORD` via env var. Token = `base64("admin:{password}")`. Sem JWT library — simples e suficiente para MVP.
- **Backend:** Vercel Serverless Functions (ES modules) em `api/admin/*.js`
- **Supabase:** `service_role` key no servidor (full access), `anon` key no frontend (RLS)
- **Upload:** Frontend converte File → base64 → POST `/api/admin/upload` → Supabase Storage bucket `hotel-images`
- **Isolamento:** Admin code via `React.lazy()` — bundle separado, erros no admin não afetam o app principal
- **Bug resolvido:** `VITE_SUPABASE_URL` no Vercel estava com o valor `VITE_SUPABASE_URL=https://...` em vez de só a URL — causava `TypeError: Invalid URL` e tela branca. Corrigido via `printf '%s'` no CLI do Vercel.
