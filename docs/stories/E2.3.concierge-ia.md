## Story
**As a** hóspede do Castro's Park Hotel,
**I want** digitar minha intenção em linguagem natural e receber recomendações personalizadas instantâneas,
**so that** eu tenha a experiência de um concierge real disponível 24h, sem precisar navegar por listas.

## Acceptance Criteria
1. Campo de busca inteligente visível na home page com placeholder convidativo
2. Endpoint `/api/concierge` (Vercel Function) recebe query do usuário e retorna top 3 recomendações
3. Resposta em ≤ 4 segundos (p95)
4. Recomendações relevantes para ≥ 80% das queries de teste (lista de 20 queries definida)
5. Cada recomendação inclui: nome do lugar, por que foi recomendado, categoria e nota
6. Fallback gracioso se LLM falhar: "Não consegui entender, tente: 'restaurante italiano perto', 'o que fazer com crianças'"
7. UI elegante e consistente com o design premium do app (integrada ao resultado de E2.1)
8. Funciona em mobile sem degradação de experiência
9. Custo por query ≤ $0.02 (monitorado)
10. Sem exposição de API keys no bundle frontend

## Dev Notes
### Arquitetura (Opção B — API Backend)
```
Frontend (ConciergeChat)
    │
    │ POST /api/concierge { query: "quero jantar italiano" }
    ▼
Vercel Function (api/concierge.js)
    │
    ├── 1. Carrega places do Supabase (ou cache)
    ├── 2. Filtra por keywords/categoria da query
    ├── 3. Envia top 10 lugares + query para LLM
    └── 4. Retorna resposta estruturada
    │
    ▼
Frontend (ConciergeResponse) — exibe top 3 cards
```

### Prompt base do Concierge
```
Você é o concierge digital do Castro's Park Hotel em Goiânia.
O hóspede pediu: "{query}"

Lugares disponíveis (top 10 mais relevantes):
{places_context}

Recomende até 3 lugares. Para cada um, responda em JSON:
{
  "places": [
    {
      "id": "...",
      "name": "...",
      "reason": "Por que recomendo (1 frase elegante)",
      "highlight": "Detalhe especial"
    }
  ],
  "message": "Mensagem introdutória curta e elegante"
}

Responda SOMENTE com JSON válido. Em português. Máximo 3 lugares.
```

### Provider LLM recomendado
- **Gemini 2.0 Flash** (Google) — rápido, barato (~$0.001/query), bom em português
- Alternativa: **Claude Haiku 4.5** — qualidade superior, custo ~$0.005/query
- Variável: `LLM_PROVIDER=gemini|claude` para facilitar troca

### Filtragem de lugares
- Busca por keyword nas colunas `name`, `category`, `subcategory`, `description`
- Filtro por `is_active = true` e `hotel_recommended = true` (prioridade)
- Enviar máximo 10 lugares no contexto para não estourar tokens

### Variáveis de ambiente a adicionar
```
LLM_API_KEY        # Chave do provider escolhido (secreta)
LLM_PROVIDER       # "gemini" ou "claude"
```

### Testing

- Verificar latência p50 e p95
- Testar fallback (query incompreensível)
