## Story
**As a** hospede do Castro's Park Hotel,
**I want** ver os eventos que estao acontecendo em Goiania durante minha estadia,
**so that** eu possa aproveitar a cidade alem dos pontos turisticos fixos.

## Acceptance Criteria
1. Pagina `/events` listando eventos ativos (is_active = true) do Supabase
2. Cada card de evento exibe: titulo, imagem, data(s), categoria, descricao resumida e link externo
3. Filtro por categoria (chips horizontais — ex: Musica, Gastronomia, Cultura, Esporte)
4. Eventos ordenados por data de inicio (mais proximos primeiro)
5. Eventos passados nao aparecem na listagem publica (start_date ou end_date >= hoje)
6. Estado vazio elegante quando nao ha eventos ("Em breve novos eventos")
7. Loading skeleton durante fetch
8. Secao "Eventos em Goiania" na home page (max 3 eventos proximos, link "Ver todos")
9. Link na navegacao (Header) para /events
10. Design consistente com o app (hotel-gold, hotel-navy, font-serif para titulos)
11. Responsivo — funciona bem em mobile

## Dev Notes

### Schema da tabela events (ja existe no Supabase)
```sql
id          uuid primary key
title       text not null
description text
image       text
address     text
link        text
start_date  date
end_date    date
category    text
is_active   boolean default true
```

### Query Supabase (hook useEvents)
```ts
// Todos os eventos ativos e futuros/em andamento
supabase
  .from('events')
  .select('*')
  .eq('is_active', true)
  .gte('end_date', new Date().toISOString().split('T')[0])
  .order('start_date', { ascending: true })
```

### Formato de data para exibicao
- Mesmo dia: "15 Mar"
- Multiplos dias: "15 — 20 Mar"
- Ano diferente do atual: "15 Mar 2027"

### Categorias esperadas (chips de filtro)
Extrair dinamicamente do banco. Mostrar "Todos" + categorias unicas dos eventos.

### Integracao na home
